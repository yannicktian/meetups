# Yannick Tian

## Role

**Staff Product AI Engineer @ Gojob**

Driving agent building and observability for the past two years.

## Track record at Gojob

- **Alpha** — Built and operated Gojob's first AI recruiter assistant in production
  - 3M+ conversations handled
  - SMS-based candidate prequalification at scale
  - Full supervision system: observability, anomaly detection, online scoring, offline evaluation, human-in-the-loop
- **Aglae SaaS — In-App Agent** — Currently building the agent that lets recruiters self-serve onboarding
  - Built on Mastra
  - RAG over similar job postings via pgvector
  - MCP server for domain logic
  - Streams structured prerequisites into a React UI
- **Kitsune (hackathon)** — Built the *Ernest* recruiter copilot in 48 hours during Gojob's 2026 hackathon
  - Full Mastra `Harness` instance per recruiting mission
  - 4 phases, 8 tools
  - SSE streaming, optimistic UI, persona swiping

## Approach

Yannick's view: **building the AI is the easy 80%. Operating it is the other 80%.** Production-grade AI requires observability, control, trust, multi-tenant isolation, multi-provider routing, error classification, and reliability layers — the boring parts that make agents trustworthy in front of real customers.
